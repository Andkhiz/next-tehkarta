# Технологические карты (ГОСТ 3.1118-82)

Веб-приложение на Next.js для создания, фильтрации и сохранения технологических карт в базу данных PostgreSQL. Данные надежно сохраняются в Docker-контейнере на жестком диске.

---

## Вариант 1. Запуск через интерфейс VS Code (Рекомендуемый)

### Шаг 1. Открытие проекта
1. Запустите **Docker Desktop**.
2. Откройте **VS Code**.
3. Нажмите `Файл (File)` ➔ `Открыть папку... (Open Folder...)` и выберите корневую папку проекта (`next-tehkarta`).

### Шаг 2. Создание файла паролей (.env)
1. На левой панели проводника VS Code зайдите в папку `web`.
2. Нажмите правой кнопкой мыши на файл `.env.example` ➔ выберите `Копировать (Copy)`.
3. Нажмите правой кнопкой мыши по пустому месту в папке `web` ➔ выберите `Вставить (Paste)`.
4. Переименуйте созданный файл ` .env.example копировать` в строго **`.env`**.

### Шаг 3. Запуск контейнеров в терминале VS Code
1. Откройте встроенный терминал: нажмите в верхнем меню `Терминал (Terminal)` ➔ `Создать терминал (New Terminal)` или используйте комбинацию клавиш **`Ctrl + ~`**.
2. Убедитесь, что терминал открыт в корне проекта, и введите команду сборки:
   ```bash
   docker-compose up -d --build
   ```
3. Дождитесь окончания сборки. Приложение сразу станет доступно в браузере по адресу: **`http://localhost:3000`**

### Шаг 4. Создание таблиц базы данных
В том же терминале VS Code примените структуру таблиц к чистой базе данных:
```bash
docker compose exec web_app npx prisma migrate deploy
```

### Шаг 5. Просмотр данных через Prisma Studio (Админка)
Если вы хотите увидеть таблицы базы данных в виде Excel-сетки, введите во встроенном терминале:
```bash
cd web
npx prisma studio
```
Админ-панель откроется в браузере по адресу: **`http://localhost:5555`**

## Вариант 2. Быстрый запуск через стандартный терминал Windows

### 1. Требования
* Установленный **Git**, **npm** и запущенный **Docker Desktop**.

### 2. Клонирование и настройка
Откройте PowerShell / CMD и выполните:
```bash
git clone https://github.com/Andkhiz/next-tehkarta.git
```

### 3. Файл окружения
Создайте файл `next/web/.env` со строкой подключения:
```bash
# Команда для PowerShell:
Copy-Item ./next-tehkarta/next/web/env.example ./next-tehkarta/next/web/.env
```

```bash
# Команда для Git Bash:
cp ./next-tehkarta/next/web/env.example ./next-tehkarta/next/web/.env
```


*Или создайте файл вручную ./next-tehkarta/next/web/.env и вставьте:*  
`DATABASE_URL="postgresql://myuser:mysecretpassword@localhost:5432/my_next_db?schema=public"`

### 4. Docker файл
В PowerShell / CMD и выполните:
```bash
cd next-tehkarta/docker
```

### 5. Старт проекта
```bash
docker-compose up -d --build
```

* Веб-интерфейс: [`http://localhost:3000`](http://localhost:3000)

## 🔄 Обновление проекта на удаленном ПК (с сохранением данных базы)

Данные технологических карт надежно хранятся на жестком диске удаленного компьютера в изолированной папке `postgres_data`. При обновлении кода через Git все сохраненные карты **полностью сохраняются**.

### Шаг 1. Отмена всех несохраненных обновлений
git reset --hard HEAD


### Шаг 2. Скачивание обновлений
Откройте терминал в корневой папке проекта на удаленном компьютере и подтяните свежую версию кода и сгенерированных SQL-миграций из репозитория:
```bash
git pull origin main
```
*Git обновит файлы интерфейса и папки миграций, не затрагивая локальную папку с базой данных.*

### Шаг 3. Создайте файла `next/web/.env` со строкой подключения:
```bash
# Команда для PowerShell:
Copy-Item ./next-tehkarta/next/web/env.example ./next-tehkarta/next/web/.env
```

```bash
# Команда для Git Bash:
cp ./next-tehkarta/next/web/env.example ./next-tehkarta/next/web/.env
```

### Шаг 4. Автоматический перезапуск и применение изменений в Docker
Перейдите в папку конфигурации Docker и запустите процесс обновления системы одной командой:
```bash
cd docker
docker compose down -v
docker compose build --no-cache
docker compose up -d
```

### ⚙️ Как это работает под капотом:
1. **База данных (`postgres_db`)** — продолжает работать в штатном режиме, либо перезапускается с полным сохранением данных из папки `postgres_data`.
2. **Контейнер миграций (`db_migrations`)** — запускается автоматически на Node 22, считывает новые файлы из папки `prisma/migrations` (пришедшие через Git) и безопасно обновляет структуру PostgreSQL. Старые данные при этом не затрагиваются. После успешного наката схем контейнер сам выключается со статусом `Exited (0)`.
3. **Веб-приложение (`web_app`)** — Docker Compose автоматически пересобирает и запускает контейнер Next.js строго после того, как контейнер миграций завершит свою работу. Приложение сразу готово к работе с обновленной структурой базы.

> 💡 **Важно:** Больше не требуется вручную заходить внутрь контейнеров и выполнять команды `exec` для накатывания таблиц. Весь процесс полностью автоматизирован на уровне манифеста Docker Compose.
