# Технологические карты (ГОСТ 3.1118-82)

Веб-приложение на Next.js для создания, фильтрации и сохранения технологических карт в базу данных PostgreSQL. Данные надежно сохраняются в Docker-контейнере на жестком диске.

---

## Вариант 1. Запуск через интерфейс VS Code (Рекомендуемый)

### Шаг 1. Открытие проекта
1. Запустите **Docker Desktop**.
2. Откройте **VS Code**.
3. Нажмите `Файл (File)` ➔ `Открыть папку... (Open Folder...)` и выберите корневую папку проекта (`next-tehkarta`).

### Шаг 2. Создание файла паролей (.env)
1. На левой панели проводника VS Code зайдите в папку `web`.
2. Нажмите правой кнопкой мыши на файл `.env.example` ➔ выберите `Копировать (Copy)`.
3. Нажмите правой кнопкой мыши по пустому месту в папке `web` ➔ выберите `Вставить (Paste)`.
4. Переименуйте созданный файл ` .env.example копировать` в строго **`.env`**.

### Шаг 3. Запуск контейнеров в терминале VS Code
1. Откройте встроенный терминал: нажмите в верхнем меню `Терминал (Terminal)` ➔ `Создать терминал (New Terminal)` или используйте комбинацию клавиш **`Ctrl + ~`**.
2. Убедитесь, что терминал открыт в корне проекта, и введите команду сборки:
   ```bash
   docker-compose up -d --build
   ```
3. Дождитесь окончания сборки. Приложение сразу станет доступно в браузере по адресу: **`http://localhost:3000`**

### Шаг 4. Создание таблиц базы данных
В том же терминале VS Code примените структуру таблиц к чистой базе данных:
```bash
docker compose exec web_app npx prisma migrate deploy
```

### Шаг 5. Просмотр данных через Prisma Studio (Админка)
Если вы хотите увидеть таблицы базы данных в виде Excel-сетки, введите во встроенном терминале:
```bash
cd web
npx prisma studio
```
Админ-панель откроется в браузере по адресу: **`http://localhost:5555`**

## Вариант 2. Быстрый запуск через стандартный терминал Windows

### 1. Требования
* Установленный **Git** и запущенный **Docker Desktop**.

### 2. Клонирование и настройка
Откройте PowerShell / CMD и выполните:
```bash
git clone https://github.com
cd next-tehkarta
```

### 3. Файл окружения
Создайте файл `web/.env` со строкой подключения:
```bash
# Команда для PowerShell:
Copy-Item ./web/.env.example ./web/.env
```
*Или создайте файл вручную и вставьте:*  
`DATABASE_URL="postgresql://myuser:mysecretpassword@localhost:5432/my_next_db?schema=public"`

### 4. Старт проекта
```bash
docker-compose up -d --build
docker compose exec web_app npx prisma migrate deploy
```
* Веб-интерфейс: `http://localhost:3000`
